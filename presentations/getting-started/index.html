VAMPD

Local Continuous Integration Made Easy

Created by the VAMPD Team

What is Continuous Integration?

"Continuous Integration (CI) is a development practice that requires developers to integrate code into a shared repository several times a day. Each check-in is then verified by an automated build, allowing teams to detect problems early.

By integrating regularly, you can detect errors quickly, and locate them more easily."

-- Thoughtworks.com

What is Vampd?

VAMPD is tool to ease continuous integration by easing and standardizing local development setup.

VAMPD is a fork of no longer existent drupal-lamp provisioner from Newmedia.

Goals of VAMPD and CI

  • Common Development Practices
  • Share Repos
  • Integrate Regularly
  • Detect Errors Quickly
  • Ease Local Development

Current Issue with CI

Developers are forgotten.

Why?

  • Different OS
  • Different Jobs
  • Server admins don't work there

How do we address this?

  • Vagrant/Virtual Box
  • Reproducible Provisioner
  • Force Testing

More importantly

Make our code reproducible!

Know what will happen

How to do that in Drupal?

  • hook_update_N
  • Features
  • CINC
  • Install profiles

What about VAMPD?

  • Supports all of this
  • Forces most of it

Install Profile


{
  "name": "lightning",
  "description": "This lightning role uses make and an install profile to install the lightning profile.",
  "chef_type": "role",
  "json_class": "Chef::Role",
  "default_attributes": {},
  "override_attributes": {
    "drupal": {
      "sites": {
        "lightning": {
          "active": true,
          "deploy": {
            "action": ["deploy", "install"]
          },
          "drush_make": {
            "api": "2",
            "files": {
              "default": "build-lightning.make"
            },
            "template": false
          },
          "drupal": {
            "settings": {
              "profile": "lightning",
              "docroot": ""
            }
          },
          "repository": {
            "host": "drupal.org",
            "uri": "http://git.drupal.org/project/lightning.git",
            "revision": "7.x-1.x"
          }
        }
      }
    }
  }
}

Exisiting Make Site With a DB to import


{
  "name": "lightning",
  "description": "This lightning role uses make and an install profile to install the lightning profile.",
  "chef_type": "role",
  "json_class": "Chef::Role",
  "default_attributes": {},
  "override_attributes": {
    "drupal": {
      "sites": {
        "lightning": {
          "active": true,
          "deploy": {
            "action": ["deploy", "import", "update"]
          },
          "drush_make": {
            "api": "2",
            "files": {
              "default": "build-lightning.make",
              "core": "drupal-org-core.make"
            },
            "template": true
          },
          "drupal": {
            "settings": {
              "profile": "lightning",
              "docroot": ""
            },
            "db_file": "/vagrant/sites/lightning/raw_db.sql"
          },
          "repository": {
            "host": "github.com",
            "uri": "git@github.com:timodwhit/lightning.git",
            "revision": "7.x-1.x"
          }
        }
      }
    }
  }
}
